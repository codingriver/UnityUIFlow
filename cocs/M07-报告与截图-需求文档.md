## 1. 模块职责

- 负责：生成 Markdown 测试报告、维护产物目录、保存步骤截图、关联失败证据与步骤时间线。
- 负责：在关键步骤和失败场景下提供可追溯的文件产物。
- 不负责：测试执行调度、不负责 Headed 面板交互、不负责 YAML 解析。
- 输入/输出：输入为 `TestResult`、`StepResult`、截图请求、输出路径；输出为 `.md` 报告文件、`.json` 结果文件、截图文件路径、产物索引。

## 2. 数据模型

### ReporterOptions（报告器配置）

| 字段名 | 类型 | 必填/可选 | 语义 | 合法范围 | 默认值 |
| --- | --- | --- | --- | --- | --- |
| reportRootPath | string | 必填 | 报告根目录 | 非空字符串 | 无 |
| screenshotRootPath | string | 必填 | 截图根目录 | 非空字符串 | 无 |
| suiteName | string | 可选 | 套件名称 | `null` 表示单用例报告 | `null` |

### StepReportEntry（步骤报告条目）

| 字段名 | 类型 | 必填/可选 | 语义 | 合法范围 | 默认值 |
| --- | --- | --- | --- | --- | --- |
| caseName | string | 必填 | 用例名称 | 非空字符串 | 无 |
| stepName | string | 必填 | 步骤名称 | 非空字符串 | 无 |
| status | string | 必填 | 步骤状态 | 仅允许 `Passed`、`Failed`、`Error`、`Skipped` | 无 |
| startedAtUtc | string | 必填 | 步骤开始时间 | ISO 8601 UTC 字符串 | 无 |
| endedAtUtc | string | 必填 | 步骤结束时间 | ISO 8601 UTC 字符串 | 无 |
| durationMs | int | 必填 | 步骤耗时 | 范围 `[0, 36000000]` | `0` |
| screenshotPath | string | 可选 | 关联截图路径（相对于报告根目录的相对路径） | `null` 表示无截图 | `null` |
| errorCode | string | 可选 | 错误码 | `null` 表示无错误 | `null` |
| errorMessage | string | 可选 | 错误消息 | `null` 表示无错误 | `null` |
| attachments | List\<string\> | 可选 | 附件路径列表 | 空列表表示无附件；单步最多 `10` 个 | 空列表 |

## 3. CRUD 操作

| 操作 | 入口 | 禁用条件 | 实现标识 | Undo语义 |
| --- | --- | --- | --- | --- |
| 请求异步截图 | 步骤 `action: screenshot` 或失败自动截图 | 输出目录为空；当前环境不允许截图 | `ScreenshotManager.CaptureAsync` `(设计提案，实现时确认)` | 不涉及；产物文件不支持 Undo |
| 请求同步截图 | 调试路径显式调用 | 当前帧上下文不可读；目标环境不允许同步截图 | `ScreenshotManager.CaptureSync` `(设计提案，实现时确认)` | 不涉及；仅调试使用 |
| 记录步骤结果 | 每步结束后 | 当前运行结果对象为空 | `MarkdownReporter.RecordStepResult` `(设计提案，实现时确认)` | 不涉及；结果日志追加写入 |
| 生成用例报告 | 用例结束后 | 输出目录不可写；用例结果为空 | `MarkdownReporter.WriteCaseReport` `(设计提案，实现时确认)` | 不涉及；若重写同名文件则覆盖旧文件 |
| 生成套件索引报告 | 套件结束后 | 套件统计为空 | `MarkdownReporter.WriteSuiteReport` `(设计提案，实现时确认)` | 不涉及；覆盖旧文件 |

## 4. 交互规格

- 触发事件：步骤结束、用例结束、套件结束、截图动作执行、失败自动抓图。
- 状态变化：`Collect step result -> Optionally capture screenshot -> Append attachment path -> Flush case report -> Flush suite report`。
- 数据提交时机：步骤结果一经确定立即写入内存缓冲；用例结束时必须落盘；套件结束时生成汇总索引。
- 取消/回退：报告生成失败不影响已完成的测试结果判定，但必须在控制台和用例结果中登记报告错误。
- 截图策略：V1 仅支持显式 `screenshot` 动作和失败自动截图；不对每一步默认截图。
- 文件命名规则：
  - 截图文件：`{caseName}-{stepIndex:D3}-{tag}-{timestamp:yyyyMMddHHmmss}.png`
  - 用例报告：`{caseName}.md`
  - 用例结果 JSON：`{caseName}.json`（供 CI 工具消费，内容为 `TestResult` 的 JSON 序列化）
  - 套件索引报告：`suite-{suiteName}.md`（`suiteName` 为 `null` 时使用 `suite-report.md`）
  - 套件结果 JSON：`suite-{suiteName}.json`

## 6. 校验规则

### 输入校验

| 规则 | 检查时机 | 级别 | 提示文案 |
| --- | --- | --- | --- |
| `reportRootPath` 与 `screenshotRootPath` 必须可创建或可写 | 初始化报告器时 | Error | `报告输出目录不可写：{path}` |
| `caseName` 不允许包含文件系统非法字符 | 生成文件名前 | Error | `测试用例名包含非法文件名字符：{caseName}` |
| 截图标签长度范围必须在 `[1, 64]` | 执行截图动作前 | Error | `截图名称非法：{name}` |
| 单张截图文件扩展名固定为 `.png` | 生成截图路径时 | Info | `截图文件格式固定为 PNG` |
| 单步骤附件数量超过 10 个时拒绝继续追加 | 添加附件时 | Warning | `步骤 {stepName} 的附件数量已达上限` |

### 错误响应

| 错误场景 | 错误码 | 错误消息模板 | 恢复行为 |
| --- | --- | --- | --- |
| 报告目录不可写 | REPORT_OUTPUT_UNAVAILABLE | `报告目录不可写：{path}` | 抛异常并将用例标记为 `Error` |
| 截图保存失败 | SCREENSHOT_SAVE_FAILED | `截图保存失败：{fileName}` | 记录错误，步骤继续按原状态结束 |
| 报告文件写入失败 | REPORT_WRITE_FAILED | `报告写入失败：{path}` | 记录错误并在控制台输出 |
| 截图请求参数非法 | SCREENSHOT_ARGUMENT_INVALID | `截图参数非法：{detail}` | 抛异常并终止当前截图步骤 |
| 附件数量超上限 | ATTACHMENT_LIMIT_EXCEEDED | `步骤 {stepName} 的附件数量超过上限` | 忽略后续附件并记录 Warning |

## 8. 技术实现要点

- 关键类与职责：
  - `ScreenshotManager` `(设计提案，实现时确认)`：负责截图请求、命名、保存与回调。
  - `MarkdownReporter` `(设计提案，实现时确认)`：负责步骤缓冲、Markdown 报告组装与落盘。
  - `JsonResultWriter` `(设计提案，实现时确认)`：负责将 `TestResult`/`TestSuiteResult` 序列化为 JSON 文件，供 CI 工具消费。
  - `ReportPathBuilder` `(设计提案，实现时确认)`：统一文件名和目录结构，负责合法化文件名字符。
  - `ArtifactIndex` `(设计提案，实现时确认)`：维护用例到附件的映射。

- Markdown 报告结构（用例级 `.md`）：

```markdown
# 测试报告：{caseName}

**状态**：{Passed|Failed|Error|Skipped}
**开始时间**：{startedAtUtc}
**结束时间**：{endedAtUtc}
**耗时**：{durationMs}ms

## 步骤详情

| 步骤 | 状态 | 耗时(ms) | 错误码 | 截图 |
| --- | --- | --- | --- | --- |
| {stepName} | {status} | {durationMs} | {errorCode 或空} | [{tag}]({relativePath}) 或空 |
```

- Markdown 套件索引结构（`suite-*.md`）：

```markdown
# 套件报告：{suiteName}

**总计**：{total} | **通过**：{passed} | **失败**：{failed} | **异常**：{errors} | **跳过**：{skipped}

## 用例汇总

| 用例 | 状态 | 耗时(ms) | 报告链接 |
| --- | --- | --- | --- |
| {caseName} | {status} | {durationMs} | [{caseName}.md]({caseName}.md) |
```

- 核心流程：

```text
Receive StepResult
-> Decide whether screenshot is needed (action=screenshot or failure + ScreenshotOnFailure)
-> Build artifact path via ReportPathBuilder
-> ScreenshotManager.CaptureAsync
-> Append row to case report buffer
-> On case finish: MarkdownReporter.WriteCaseReport + JsonResultWriter.WriteCaseJson
-> On suite finish: MarkdownReporter.WriteSuiteReport + JsonResultWriter.WriteSuiteJson
```

- 性能约束：
  - 截图必须异步保存，禁止在主流程中阻塞超过 `1` 帧。
  - 报告写入按用例落盘，禁止每个步骤都重写整份 Markdown 文件。
  - 套件汇总只在套件结束时生成 1 次。
  - 单用例 Markdown 缓冲上限为 `10000` 行，超出时报 `REPORT_WRITE_FAILED`。

- 禁止项：
  - 禁止把二进制截图内容内联写入 Markdown。
  - 禁止依赖绝对临时目录，所有产物必须落在配置目录下。
  - Markdown 中的截图链接必须使用相对路径，不使用绝对路径。

- TODO(待确认)：Editor 模式下异步截图落盘完成判定方式需原型验证（见 `TODO-007`）。

## 9. 验收标准

1. [用例包含显式 `screenshot` 步骤] -> [执行用例] -> [在截图目录生成 PNG 文件，并在 Markdown 中写入相对链接]
2. [步骤执行失败且 `ScreenshotOnFailure=true`] -> [执行用例] -> [自动保存失败截图，并在失败步骤行关联该附件]
3. [用例执行完成] -> [查看报告目录] -> [生成 1 份 `{caseName}.md` 和 1 份 `{caseName}.json` 报告]
4. [套件包含 2 个用例] -> [执行套件] -> [生成套件索引报告 `suite-report.md` 与 `suite-report.json`，并准确汇总通过/失败/跳过计数]
5. [输出目录不可写] -> [执行用例] -> [返回 `REPORT_OUTPUT_UNAVAILABLE`，不得静默吞错]
6. [截图文件名包含特殊字符] -> [生成截图路径] -> [`ReportPathBuilder` 自动替换非法字符为 `_`，不报错]

## 10. 边界规范

- 空数据：步骤没有截图、错误或附件时，Markdown 中对应列留空，不写占位字符串 `null`。
- 单元素：单步骤单截图的用例也必须生成完整报告，且附件链接可点击。
- 上下限临界值：单步骤附件上限为 `10`；第 `10` 个合法，第 `11` 个拒绝追加并记录 `ATTACHMENT_LIMIT_EXCEEDED`。
- 异常数据恢复：截图失败时不改变原步骤状态，但必须写入可追溯错误消息到步骤报告行。

## 11. 周边可选功能

- P1：支持 HTML 报告导出；当前预留 `IReportRenderer` 接口（`void Render(TestResult result, string outputPath)`），默认实现为 Markdown，后续可注入 HTML 实现。
- P1：支持截图压缩与缩略图；当前预留 `ScreenshotPostProcessor` 接口（`byte[] Process(byte[] raw)`），默认直接落盘原始截图。
- P2：支持视频录屏附件；当前不预留视频编码链路。
