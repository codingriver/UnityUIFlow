## 1. 模块职责

- 负责：解析并执行 `#id`、`.class`、`Type`、`[attribute=value]`、后代选择器、直接子选择器等元素定位规则。
- 负责：在指定根节点下完成同步查找、等待查找、可见性判定和定位失败诊断。
- 不负责：实际点击、输入、滚动等动作派发；不负责 YAML 解析。
- 输入/输出：输入为选择器字符串或已编译选择器（`SelectorExpression`）、根 `VisualElement`、超时时间（`WaitOptions`）；输出为目标 `VisualElement`（`FindResult`）或标准化异常。

## 2. 数据模型

### SelectorExpression（已编译选择器，与 M02 共享定义）

| 字段名 | 类型 | 必填/可选 | 语义 | 合法范围 | 默认值 |
| --- | --- | --- | --- | --- | --- |
| raw | string | 必填 | 原始选择器文本 | 非空字符串，长度范围 `[1, 256]` | 无 |
| segments | List\<SelectorSegment\> | 必填 | 分段后的选择器节点 | 列表长度范围 `[1, 16]` | 无 |

### SelectorSegment（选择器分段）

| 字段名 | 类型 | 必填/可选 | 语义 | 合法范围 | 默认值 |
| --- | --- | --- | --- | --- | --- |
| combinator | string | 必填 | 连接符 | 仅允许 `Descendant`、`Child`、`Self` | `Self` |
| tokenType | string | 必填 | token 类型 | 仅允许 `Id`、`Class`、`Type`、`Attribute`、`Pseudo` | 无 |
| tokenValue | string | 必填 | token 值 | 非空字符串 | 无 |
| pseudoArguments | string | 可选 | 伪类参数（如 `nth-child` 的 `n`） | `null` 表示无参数；V1 未使用 | `null` |

### WaitOptions（等待查找选项）

| 字段名 | 类型 | 必填/可选 | 语义 | 合法范围 | 默认值 |
| --- | --- | --- | --- | --- | --- |
| timeoutMs | int | 必填 | 等待查找超时 | 范围 `[100, 600000]`，包含端点 | 无 |
| pollIntervalMs | int | 必填 | 轮询间隔 | 范围 `[16, 1000]`，包含端点 | `16` |
| requireVisible | bool | 必填 | 是否要求元素可见 | `true` 或 `false` | `true` |

### FindResult（查找结果）

| 字段名 | 类型 | 必填/可选 | 语义 | 合法范围 | 默认值 |
| --- | --- | --- | --- | --- | --- |
| element | VisualElement | 可选 | 命中元素 | `null` 表示未命中 | `null` |
| foundAtMs | int | 可选 | 命中时耗时 | `null` 表示未命中；范围 `[0, timeoutMs]` | `null` |

## 3. CRUD 操作

| 操作 | 入口 | 禁用条件 | 实现标识 | Undo语义 |
| --- | --- | --- | --- | --- |
| 同步查找元素 | `ElementFinder.Find(SelectorExpression selector, VisualElement root)` | `selector` 为空；`root` 为 `null` | `ElementFinder.Find` `(设计提案，实现时确认)` | 不涉及；只读操作 |
| 等待元素出现 | `ElementFinder.WaitForElement(SelectorExpression selector, VisualElement root, WaitOptions options)` | `options.timeoutMs` 超界；`root` 为 `null` | `ElementFinder.WaitForElement` `(设计提案，实现时确认)` | 不涉及；只读操作 |
| 解析选择器 | `SelectorCompiler.Compile(string selector)` | 选择器语法错误 | `SelectorCompiler.Compile` `(设计提案，实现时确认)` | 不涉及；只读操作 |
| 判定元素可见性 | 所有断言与默认查找流程 | `element` 为 `null` | `ElementVisibilityEvaluator.IsVisible` `(设计提案，实现时确认)` | 不涉及；只读操作 |
| 枚举候选元素 | 复杂选择器匹配过程中自动触发 | 根树已释放；节点已销毁 | `VisualElementQueryAdapter.QueryAll` `(设计提案，实现时确认)` | 不涉及；只读操作 |

## 4. 交互规格

- 触发事件：动作执行前对 `selector` 发起同步查找或等待查找。
- 状态变化：`Compile selector -> Enumerate candidates -> Filter by segments -> Evaluate visibility -> Return element or timeout`。
- 数据提交时机：同步查找立即返回结果；等待查找在命中元素或达到 `timeoutMs` 时结束。
- 取消/回退：等待期间若运行被停止，则立即退出等待并上抛中止错误，不继续轮询。
- 可见性规则：默认要求 `resolvedStyle.display != DisplayStyle.None`、`resolvedStyle.visibility != Visibility.Hidden`、`resolvedStyle.opacity > 0`，并且元素未脱离面板（`panel != null`）。细节见 `TODO(待确认)`。
- 重复命中：同一选择器匹配多个元素时，V1 固定返回文档顺序中的第一个元素。
- 查找策略（设计提案，实现时确认）：优先调用 UIToolkit 内置 `UQuery` API（`root.Q<VisualElement>(name: "...", className: "...")`、`root.Query<VisualElement>().ToList()`）完成简单 ID/Class/Type 查找；仅当选择器包含后代组合符、子组合符或属性过滤时，才回退到 `VisualElementQueryAdapter` 自定义遍历。这样可减少自定义代码维护量并保持与 UIToolkit 版本的兼容性。

## 6. 校验规则

### 输入校验

| 规则 | 检查时机 | 级别 | 提示文案 |
| --- | --- | --- | --- |
| `selector` 不能为空串 | 编译选择器前 | Error | `选择器不能为空` |
| `root` 不能为空 | 查找前 | Error | `根节点不能为空` |
| `timeoutMs` 必须在允许范围内 | 进入等待查找前 | Error | `等待超时值非法：{timeoutMs}` |
| `pollIntervalMs` 最小值为 `16` | 创建 `WaitOptions` 时 | Error | `轮询间隔不能小于 16ms` |
| 属性选择器只允许 `name`、`tooltip`、`data-*` | 编译属性选择器时 | Error | `属性选择器 {attribute} 不受支持` |
| `>` 不能出现在选择器开头或结尾 | 编译选择器时 | Error | `选择器 {selector} 的子级连接符位置非法` |
| `:first-child` 仅允许用于匹配结果裁剪 | 编译伪类时 | Warning | `伪类 {pseudo} 仅支持 first-child` |

### 错误响应

| 错误场景 | 错误码 | 错误消息模板 | 恢复行为 |
| --- | --- | --- | --- |
| 选择器为空 | SELECTOR_EMPTY | `选择器不能为空` | 抛异常并终止当前步骤 |
| 选择器语法错误 | SELECTOR_SYNTAX_INVALID | `选择器语法错误：{selector}` | 抛异常并终止当前步骤 |
| 根节点不存在 | ROOT_ELEMENT_MISSING | `根节点不存在，无法查找 {selector}` | 抛异常并终止当前步骤 |
| 等待超时 | ELEMENT_WAIT_TIMEOUT | `等待元素超时：{selector}，超时 {timeoutMs}ms` | 抛异常并将当前步骤记为失败 |
| 元素不可见 | ELEMENT_NOT_VISIBLE | `元素不可见：{selector}` | 抛异常并将当前步骤记为失败 |
| 查询期间元素已释放 | ELEMENT_DISPOSED_DURING_QUERY | `元素树已释放：{selector}` | 抛异常并终止当前步骤 |

## 8. 技术实现要点

- 关键类与职责：
  - `ElementFinder` `(设计提案，实现时确认)`：统一同步/等待查找接口；简单选择器走 `UQuery`，复杂选择器走自定义遍历。
  - `SelectorCompiler` `(设计提案，实现时确认)`：将字符串选择器拆解为 `SelectorExpression`。
  - `ElementVisibilityEvaluator` `(设计提案，实现时确认)`：统一可见性判断逻辑，不分散在各动作中。
  - `VisualElementQueryAdapter` `(设计提案，实现时确认)`：封装 `UQuery` 调用与自定义遍历，屏蔽 API 差异。

- 核心流程（含查找策略分支）：

```text
Receive SelectorExpression
-> If segments.Count == 1 and tokenType in [Id, Class, Type]
   -> Use root.Q<VisualElement>(name/className/type) via UQuery (fast path)
-> Else
   -> VisualElementQueryAdapter.QueryAll: traverse + match tokens + apply combinators
-> Apply visibility filter when requireVisible = true
-> Return first matched element or throw ELEMENT_WAIT_TIMEOUT
```

- 性能约束：
  - 同步查找目标耗时不超过 `10ms`，基线为 `2000` 节点以内的 UI 树。`TODO(需原型验证)：此性能目标需在 PoC 中实测确认，见 TODO-012。`
  - 等待查找轮询间隔最小 `16ms`（约 1 帧），禁止 `0ms` busy loop。
  - 同一步骤内重复使用同一选择器时，允许缓存编译结果，不缓存 `VisualElement` 实例。
  - 禁止通过全局静态变量保存上一次找到的 `VisualElement`。

- TODO(待确认)：UIToolkit 中"可见"的最终判定是否需要同时考虑 `pickingMode`、`enabledInHierarchy` 与裁剪区域，需原型验证（见 `TODO-011`）。

## 9. 验收标准

1. [根节点下存在 `name=login-button` 的元素] -> [调用 `Find("#login-button", root)`] -> [返回该元素实例]
2. [根节点下存在 `.panel > .btn` 结构] -> [调用子级选择器查找] -> [仅匹配直接子级按钮，不匹配孙级]
3. [目标元素在 2 秒后创建] -> [调用 `WaitForElement(selector, root, 5000ms)`] -> [在 5000ms 内成功返回元素]
4. [目标元素始终不存在] -> [调用 `WaitForElement(selector, root, 1000ms)`] -> [抛出 `ELEMENT_WAIT_TIMEOUT`]
5. [同一选择器匹配多个元素] -> [调用 `Find`] -> [返回树顺序中的第一个匹配项]
6. [选择器为简单 `#btn`] -> [调用 `Find`] -> [返回正确元素，执行耗时 < 5ms（基线 2000 节点 UI 树）]

## 10. 边界规范

- 空数据：空字符串选择器一律报 `SELECTOR_EMPTY`，不得尝试返回根节点。
- 单元素：只有根节点自身可匹配时，`Self` 选择器必须允许命中根节点。
- 上下限临界值：`timeoutMs=100` 和 `600000` 合法；`pollIntervalMs=16` 和 `1000` 合法；`pollIntervalMs<16` 报 `Error`。
- 异常数据恢复：等待期间根节点或面板被销毁时，立即抛出 `ELEMENT_DISPOSED_DURING_QUERY`，不继续轮询。

## 11. 周边可选功能

- P1：支持 `:nth-child(n)` 伪类；当前 `SelectorSegment.pseudoArguments` 字段已预留，V1 始终为 `null`，后续解析器读取该字段实现序号匹配。
- P1：支持元素列表断言（`FindAll` 返回全部匹配）；当前预留 `ElementFinder.FindAll` 扩展签名，不进入 V1 主流程。
- P2：支持基于正则的文本内容匹配；当前不预留额外 DSL 语法，扩展时在 `SelectorSegment.tokenType` 中新增 `TextContains` 值。
