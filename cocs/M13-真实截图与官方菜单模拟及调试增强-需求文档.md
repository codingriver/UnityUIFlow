# M13 真实截图与官方菜单模拟及调试增强 需求文档

版本：1.3.0
日期：2026-04-13
状态：以当前代码实现为准更新

---

## 1. 模块职责

- 负责补齐 `ScreenshotManager` 的真实截图能力，不再把 2x2 占位 PNG 当作正常输出。
- 负责将 `ContextMenuSimulator`、`PopupMenuSimulator` 挂接到现有官方 `EditorWindowPanelSimulator` 宿主链路。
- 负责让 `DebugOnFailure` 在运行控制链中真实生效，而不是只停留在配置字段。
- 负责为 `click`、`double_click`、`drag`、`hover` 增加 `button` 与 `modifiers` 语义。
- 负责与 M05 / M12 对齐菜单、命令、修饰键和官方宿主链路的实现边界。
- 不负责新增 YAML 语法体系，不负责像素级视觉 diff，不负责系统级 IME / 剪贴板 / 多窗口协同。

## 2. 数据模型

### ScreenshotManager

| 字段名 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| options | `TestOptions` | 是 | 截图输出目录与行为配置 |
| windowProvider | `Func<EditorWindow>` | 否 | 优先截图的宿主窗口提供器 |

### 动作扩展参数

| 动作 | 参数 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| `click` | `button` | `string` | `left` | `left/right/middle` |
| `click` | `modifiers` | `string` | `none` | `shift/ctrl/alt/cmd`，可组合 |
| `double_click` | `button` | `string` | `left` | 同上 |
| `double_click` | `modifiers` | `string` | `none` | 同上 |
| `drag` | `button` | `string` | `left` | 同上 |
| `drag` | `modifiers` | `string` | `none` | 同上 |
| `hover` | `modifiers` | `string` | `none` | 鼠标移动时附带修饰键 |
| `select_context_menu_item` | `item` | `string` | 无 | 要选择的上下文菜单项文本 |
| `select_popup_menu_item` | `item` | `string` | 无 | 要选择的弹出菜单项文本 |

## 3. CRUD 操作

| 操作 | 入口 | 输出 |
| --- | --- | --- |
| 真实截图 | `ScreenshotManager.CaptureSync/CaptureAsync` | 当前窗口或屏幕 PNG |
| 打开上下文菜单 | `open_context_menu` | 菜单进入已打开状态 |
| 选择上下文菜单项 | `select_context_menu_item` | 执行目标菜单 action |
| 打开弹出菜单 | `open_popup_menu` | 菜单进入已打开状态 |
| 选择弹出菜单项 | `select_popup_menu_item` | 执行目标菜单 action |
| 菜单断言 | `assert_menu_item` / `assert_menu_item_disabled` | 校验菜单项状态 |
| 失败暂停 | `DebugOnFailure=true` 且 `Headed=true` | 运行控制器暂停等待人工继续 |

## 4. 交互规格

- `ScreenshotManager` 当前优先按以下顺序截图：
  1. `windowProvider` 返回的宿主窗口
  2. `EditorWindow.focusedWindow`
  3. `ScreenCapture.CaptureScreenshotAsTexture()` 全屏回退
- `open_context_menu` 走官方菜单链路：
  - 先清空旧菜单状态
  - 通过 `PanelSimulator.Click(..., RightMouse, modifiers)` 打开菜单
  - 通过 `ContextMenuSimulator` 读取与选择当前菜单项
- `open_popup_menu` 走官方菜单链路：
  - 先清空旧菜单状态
  - 优先点击 `unity-base-popup-field__input` 触发区域
  - 通过 `PopupMenuSimulator` 读取与选择当前菜单项
- `DebugOnFailure` 当前行为：
  - 步骤失败且 `Headed=true`、`DebugOnFailure=true` 时，执行链调用 `RuntimeController.PauseForFailure()`
  - 恢复后再继续清理流程，而不是直接吞掉暂停状态

---

## 5. 视觉规格

- 截图产物必须来自真实像素，不再把 2x2 占位纹理视作成功截图。
- 菜单与调试相关示例窗口当前已具备以下命名目标：
  - `context-menu-target`
  - `popup-menu-dropdown`
  - `pointer-status`
  - `menu-status`

## 6. 校验规则

| 规则 | 级别 | 处理 |
| --- | --- | --- |
| `button` 非 `left/right/middle` | Error | 抛出 `ACTION_PARAMETER_INVALID` |
| `modifiers` 包含未知标记 | Error | 抛出 `ACTION_PARAMETER_INVALID` |
| 菜单动作缺少 `item` | Error | 抛出 `ACTION_PARAMETER_MISSING` |
| 官方菜单链路不可用时执行菜单动作 | Error | 抛出 `OFFICIAL_UI_TEST_FRAMEWORK_UNAVAILABLE` |
| 菜单已打开但目标项不存在 | Error | 抛出 `ACTION_EXECUTION_FAILED` |

| 模块 | 联动方式 |

## 8. 技术实现要点

- 当前真实截图实现：
  - 优先走编辑器窗口或屏幕像素抓取
  - 真实链路失败时再回退到兼容方案
  - 不再默认输出占位 PNG
- 当前官方菜单桥实现：
  - 复用 `EditorWindowPanelSimulator`
  - 通过 `ContextMenuSimulator` / `PopupMenuSimulator` 读取菜单状态
  - 通过 `UnityUIFlowSimulationSession` 维护菜单生命周期
- 当前指针增强实现：
  - `ActionHelpers.ParseMouseButton`
  - `ActionHelpers.ParseEventModifiers`
  - 官方链路优先走 `PanelSimulator`
  - fallback 链路同步透传按钮与修饰键

## 9. 验收标准

1. [执行 `screenshot`] -> [输出 PNG] -> [图片宽高大于 2]
2. [执行 `click` 传入 `button=right`、`modifiers=shift,ctrl`] -> [示例窗口读取状态] -> [能看到右键与修饰键信息]
3. [执行 `drag` 传入 `button=right`、`modifiers=alt`] -> [示例窗口读取状态] -> [拖拽完成且能看到对应状态]
4. [执行 `open_context_menu` + `select_context_menu_item`] -> [示例窗口状态写入] -> [`Context: ...` 更新]
5. [执行 `open_popup_menu` + `select_popup_menu_item`] -> [示例窗口状态写入] -> [`Popup: ...` 更新]
6. [失败发生且 `DebugOnFailure=true`] -> [运行控制器暂停] -> [人工恢复后继续清理]

## 10. 边界规范

- 当前 `drag_reorder` 已实现，但边界是 `ListView.itemsSource` 级别逻辑重排，不等价于跨窗口、跨面板、文件拖放。
- 当前菜单动作只覆盖官方 `com.unity.ui.test-framework` 已暴露的 EditorWindow / panel 场景。
- 当前截图优先面向当前 EditorWindow 或编辑器屏幕，不承诺多显示器精确裁剪。
- `modifiers` 当前支持：
  - `shift`
  - `ctrl` / `control`
  - `alt`
  - `cmd` / `command` / `meta`
- 仍未覆盖：
  - `ObjectField` / `DragAndDrop`
  - 多窗口拖放
  - IME / 系统剪贴板
  - 像素级视觉 diff
