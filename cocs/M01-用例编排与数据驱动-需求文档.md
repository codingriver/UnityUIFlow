## 1. 模块职责

- 负责：定义 YAML 测试用例的文件结构、fixture 结构、数据驱动入口、变量替换范围和步骤列表约束。
- 负责：把 QA 可编辑的声明式文本约束为稳定的数据输入契约，供 `YamlTestCaseParser` 消费。
- 不负责：YAML 语法解析、不负责执行动作、不负责元素定位、不负责生成测试报告。
- 输入/输出：输入为 `.yaml` 文本文件或外部 CSV/JSON 数据文件；输出为 `TestCaseDefinition`、`FixtureDefinition`、`DataSourceDefinition` 的逻辑契约。

## 2. 数据模型
| 字段名 | 类型 | 必填/可选 | 语义 | 合法范围 | 默认值 |
| --- | --- | --- | --- | --- | --- |
| name | string | 必填 | 用例名称 | 非空字符串，长度范围 `[1, 120]` | 无 |
| description | string | 可选 | 用例说明 | `null` 表示无描述；长度范围 `[0, 500]` | `""` |
| fixture | FixtureDefinition | 可选 | 用例级 setup/teardown 配置 | `null` 表示无 fixture | `null` |
| data | DataSourceDefinition | 可选 | 数据驱动配置 | `null` 表示单次执行；仅允许内联列表、CSV、JSON 三种来源 | `null` |
| steps | List<StepDefinition> | 必填 | 主步骤列表 | 列表长度范围 `[1, 500]` | 无 |
| sourceFile | string | 必填 | YAML 源文件相对路径 | 以 `Assets/` 或项目内相对路径表示；非空 | 无 |
| tags | List<string> | 可选 | 过滤标签 | 空列表表示无标签；单标签长度范围 `[1, 32]` | 空列表 |
| timeoutMs | int | 可选 | 用例级默认超时覆盖值 | 范围 `[100, 600000]`，包含端点；`null` 表示沿用 `TestOptions.DefaultTimeoutMs` | `null` |
| setup | List<StepDefinition> | 可选 | fixture 前置步骤 | 空列表表示无前置步骤；长度范围 `[0, 100]` | 空列表 |
| teardown | List<StepDefinition> | 可选 | fixture 清理步骤 | 空列表表示无清理步骤；长度范围 `[0, 100]` | 空列表 |
| rows | List<Dictionary<string, string>> | 可选 | 内联数据行 | 空列表表示无数据；每行键名必须唯一 | 空列表 |
| from_csv | string | 可选 | CSV 数据源路径 | `null` 表示未使用；路径后缀必须为 `.csv` | `null` |
| from_json | string | 可选 | JSON 数据源路径 | `null` 表示未使用；路径后缀必须为 `.json` | `null` |
| name | string | 可选 | 步骤名 | `null` 表示执行时回退为 `action` 名称；长度范围 `[0, 120]` | `null` |
| action | string | 必填 | 步骤动作标识 | `snake_case` 或小写字符串，长度范围 `[1, 64]` | 无 |
| selector | string | 可选 | 元素选择器 | `null` 表示该步骤不依赖元素定位 | `null` |
| value | string | 可选 | 输入值 | `null` 表示无值；允许 `{{ variable }}` 模板 | `null` |
| expected | string | 可选 | 断言期望值 | `null` 表示该动作不使用期望值；允许 `{{ variable }}` 模板 | `null` |
| timeout | string | 可选 | 步骤级时长字面量 | `null` 表示沿用用例默认超时；允许格式 `Xs`、`Xms`，其中 `X` 范围 `[1, 600000]` | `null` |
| duration | string | 可选 | 等待时长字面量 | `null` 表示该动作不使用等待时长；允许格式 `Xs`、`Xms`，其中 `X` 范围 `[1, 600000]` | `null` |
| if | ConditionDefinition | 可选 | 条件执行定义 | `null` 表示无条件执行；V1 仅允许 `exists` 条件 | `null` |
| repeat_while | LoopDefinition | 可选 | 循环定义 | `null` 表示非循环步骤；V1 仅允许 `exists` 条件 | `null` |

## 3. CRUD 操作
| 操作 | 入口 | 禁用条件 | 实现标识 | Undo语义 |
| --- | --- | --- | --- | --- |
| 读取 YAML 用例 | `TestRunner.RunTest(string yamlPath, TestOptions options)` 调用流程 | `yamlPath` 为空、文件不存在、后缀不是 `.yaml` | `YamlTestCaseParser.ParseFile` `(设计提案，实现时确认)` | 不涉及；只读操作 |
| 展开 fixture 步骤 | 执行引擎进入用例前后生命周期 | `fixture` 为 `null` 或对应步骤列表为空 | `ExecutionPlanBuilder.AppendFixtureSteps` `(设计提案，实现时确认)` | 不涉及；只读操作 |
| 加载数据驱动行 | `data` 字段声明为内联、CSV、JSON | 同时声明多种数据源；数据文件不存在；数据列不齐 | `TestDataResolver.ResolveRows` `(设计提案，实现时确认)` | 不涉及；只读操作 |
| 渲染变量模板 | 每步执行前对 `value`、`expected`、`selector`、`name` 做模板替换 | 变量不存在；变量值为 `null`；模板语法未闭合 | `TemplateRenderer.Render` `(设计提案，实现时确认)` | 不涉及；失败时回退为用例失败 |
| 校验步骤结构 | YAML 解析完成后进入执行计划编译前 | `steps` 为空；步骤同时声明 `action` 与 `repeat_while` 冲突字段 | `TestCaseSchemaValidator.Validate` `(设计提案，实现时确认)` | 不涉及；失败时阻断执行 |

## 4. 交互规格

- 触发事件：调用 `TestRunner.RunTest` 或 `TestRunner.RunSuite` 后开始读取 YAML 文件。
- 状态变化：先构建 `TestCaseDefinition`，再解析 `fixture`，再扩展 `data` 行，最后产出逐步展开后的逻辑步骤序列。
- 数据提交时机：YAML 内容在解析成功后一次性提交给执行计划编译模块；模板变量在每次数据行执行前重新渲染，不做跨行缓存。
- 取消/回退：任一步骤结构校验失败时，中止当前用例；若失败发生在 `setup` 之后，仍必须尝试执行 `teardown`。
- 条件执行：当 `if.exists` 指定的选择器存在时执行当前步骤；不存在时跳过当前步骤并记录为 `Skipped`。
- 循环执行：`repeat_while.condition` 每轮开始前重新求值；当条件首次为 `false` 时退出循环；单循环最大迭代次数固定为 `1000`，超出时报错。

## 6. 校验规则
### 输入校验
| 规则 | 检查时机 | 级别 | 提示文案 |
| --- | --- | --- | --- |
| `name` 必须存在且非空 | 解析 YAML 顶层字段后 | Error | `测试用例名称不能为空` |
| `steps` 列表至少包含 1 个步骤 | 解析主步骤列表后 | Error | `测试用例 {name} 至少需要 1 个步骤` |
| `data` 只能选择一种来源 | 解析 `data` 字段后 | Error | `测试用例 {name} 的 data 只能声明一种数据源` |
| CSV 表头必须与所有数据行列数一致 | 读取 CSV 后 | Error | `数据文件 {path} 的列数不一致` |
| JSON 根节点必须是对象数组 | 读取 JSON 后 | Error | `数据文件 {path} 必须是对象数组` |
| 模板变量必须能从当前数据行解析 | 每步执行前渲染模板时 | Error | `步骤 {stepName} 缺少变量 {variable}` |
| `timeout` 与 `duration` 必须是合法时长字面量 | 编译执行计划时 | Error | `步骤 {stepName} 的时长格式非法` |
| `repeat_while` 不能与 `selector`、`value` 同级混用 | 步骤结构校验时 | Error | `步骤 {stepName} 的 repeat_while 结构非法` |

### 错误响应
| 错误场景 | 错误码 | 错误消息模板 | 恢复行为 |
| --- | --- | --- | --- |
| YAML 文件路径为空或后缀错误 | TEST_CASE_PATH_INVALID | `测试用例路径非法：{path}` | 抛异常并终止当前用例 |
| YAML 文件不存在 | TEST_CASE_FILE_NOT_FOUND | `测试用例文件不存在：{path}` | 抛异常并终止当前用例 |
| YAML 顶层结构缺字段 | TEST_CASE_SCHEMA_INVALID | `测试用例 {name} 结构非法：{detail}` | 抛异常并终止当前用例 |
| 数据文件不存在 | TEST_DATA_FILE_NOT_FOUND | `数据文件不存在：{path}` | 抛异常并终止当前用例 |
| 数据行缺少变量 | TEST_DATA_VARIABLE_MISSING | `数据行缺少变量 {variable}` | 抛异常并终止当前数据行与当前用例 |
| 循环次数超过上限 | TEST_LOOP_LIMIT_EXCEEDED | `步骤 {stepName} 超过最大循环次数 {limit}` | 抛异常并终止当前用例 |

## 8. 技术实现要点

- 关键类与职责：
  - `TestCaseDefinition` `(设计提案，实现时确认)`：承载单个 YAML 用例的内存模型。
  - `StepDefinition` `(设计提案，实现时确认)`：承载动作步骤、条件步骤与循环步骤。
  - `TestDataResolver` `(设计提案，实现时确认)`：解析内联、CSV、JSON 数据源并输出统一行模型。
  - `TemplateRenderer` `(设计提案，实现时确认)`：对 `{{ variable }}` 占位符做字符串替换。
- 核心流程：
```text
Parse YAML text
-> Validate top-level schema
-> Resolve fixture/setup/teardown
-> Resolve data rows
-> For each data row
-> Clone steps
-> Render templates
-> Output normalized case data to ExecutionPlanBuilder
```
- 性能约束：
  - 单个 YAML 文件只允许读取一次源文本。
  - 数据驱动展开后单用例总步骤数上限为 `5000`，超出时报错。
  - 禁止在每步执行时重新读取 CSV 或 JSON 文件。
  - 必须对模板替换使用当前数据行字典访问，禁止反射遍历对象。
- TODO(待确认)：CSV 编码兼容范围仅确定需要支持 UTF-8；是否同时支持带 BOM 的 UTF-8、GB18030 需原型验证。

## 9. 验收标准
1. [存在合法 YAML 文件，且 `steps` 为 3 条] -> [调用 `RunTest`] -> [成功生成 1 个 `ExecutionPlan`，步骤数等于 3]
2. [YAML 中配置 `setup`、`steps`、`teardown`] -> [执行用例] -> [执行顺序固定为 `setup -> steps -> teardown`]
3. [YAML 中配置 2 行 `data`] -> [执行用例] -> [同一逻辑步骤被展开执行 2 次，且模板变量分别替换为两行数据]
4. [步骤包含 `if.exists`，目标元素不存在] -> [执行用例] -> [该步骤结果为 `Skipped`，后续步骤继续执行]
5. [步骤包含 `repeat_while`，条件第 3 次评估为 false] -> [执行用例] -> [循环体准确执行 2 次后退出]
6. [YAML 缺少 `name`] -> [执行解析] -> [抛出 `TEST_CASE_SCHEMA_INVALID`，不进入执行阶段]

## 10. 边界规范

- 空数据：`data` 为 `null` 时，执行引擎按单次执行处理，不创建空数据行占位对象。
- 单元素：`steps` 仅有 1 步时，必须允许正常执行，不要求 fixture、description、tags 同时存在。
- 上下限临界值：`timeoutMs=100` 与 `timeoutMs=600000` 均合法；小于 `100` 或大于 `600000` 一律报错。
- 异常数据恢复：CSV 或 JSON 数据文件解析失败时，不得回退到“无数据模式”；必须立即终止当前用例，避免误跑错误数据。

## 11. 周边可选功能

- P1：支持 `data.from_jsonl` 行式 JSON 数据源；当前不预留字段，后续通过 `DataSourceDefinition` 扩展。
