# UnityUIFlow 开发路线图与交付物

版本：1.0.0  
日期：2026-04-08  
状态：补充版

## 里程碑

| 里程碑 | 时间 | 目标 | 关键验收点 |
| --- | --- | --- | --- |
| M0 PoC | 第 1 周 | 验证测试宿主、最小 YAML、最小动作闭环 | 能在 Editor 中打开窗口、读取 YAML、执行 1 个 click 动作 |
| M1 核心引擎 | 第 2-3 周 | 补齐解析、执行、动作、定位主链路 | `M01-M05` 主流程全部可跑通 |
| M2 Headed 调试 | 第 4 周 | 完成可视化调试面板 | 支持运行、暂停、步进、高亮、失败暂停 |
| M3 报告与集成 | 第 5 周 | 完成截图、Markdown、CLI、CI | 批处理执行可产出报告和标准退出码 |
| M4 文档与示例 | 第 6 周 | 完成样例、用户手册、API 文档 | 提供示例项目、API 文档、CI 示例 |

## 每阶段交付物

| 阶段 | 必交付物 |
| --- | --- |
| M0 | 最小原型代码、原型记录、依赖确认结果 |
| M1 | YAML 解析器、执行引擎、动作注册表、元素定位器、Fixture 基座 |
| M2 | Headed 面板、高亮实现、运行控制器 |
| M3 | Markdown 报告器、截图管理器、CLI 解析器、CI 示例脚本 |
| M4 | 示例 YAML 用例、Page Object 示例、开发者手册、DocFX API 文档 |

## 完成定义

| 项目 | 完成标准 |
| --- | --- |
| 功能完成 | 对应 `MXX` 需求文档中的验收标准全部可执行且通过 |
| 文档完成 | `cocs` 中模块文档、总览文档、TODO 保持同步 |
| 示例完成 | 至少提供 5 个典型 YAML 用例 |
| 工程完成 | 可在本地 Editor 和 CI 无头模式运行 |
| 风险完成 | `TODO.md` 中 P0 项全部关闭 |

## 首版交付物清单

1. UnityUIFlow 源码仓库。
2. 示例项目或示例场景。
3. 5 个典型 YAML 用例。
4. C# 自定义动作示例。
5. Page Object 示例。
6. Markdown 报告样例。
7. GitHub Actions 或 GitLab CI 配置样例。
8. API 文档导出结果。

## 风险关口

| 风险点 | 必须在何时关闭 |
| --- | --- |
| Unity 版本统一 | M0 结束前 |
| `com.unity.ui.test-framework` 版本与可用性 | M0 结束前 |
| EditMode 协程调度可行性 | M1 开始前 |
| Headed 高亮叠加方案 | M2 开始前 |
| 异步截图落盘判定 | M3 开始前 |

## 发布建议

1. 先以内网仓库或源码包发布首版。
2. 首版不打 `.unitypackage` 和源码仓库双制式并行，优先源码仓库。
3. 对外发布前先冻结 YAML schema 和动作名集合。
