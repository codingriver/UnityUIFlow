# UnityUIFlow 架构设计与技术约束

版本：1.0.0  
日期：2026-04-08  
状态：补充版

## 项目背景

UnityUIFlow 的目标是为 UIToolkit 提供一套接近 Playwright headed 模式体验的 Editor 内自动化测试框架。

V1 聚焦 `EditMode + EditorWindow + UIToolkit` 场景，不覆盖 uGUI、真机移动端和录制回放。

## 产品目标

| 目标 | 结论 |
| --- | --- |
| 声明式测试用例 | 必须支持 YAML 手写用例，由 QA 独立维护 |
| 真实交互模拟 | 必须基于官方 UI 测试能力模拟点击、输入、拖拽、滚动、悬停 |
| Headed 调试体验 | 必须支持实时高亮、步进、失败暂停、现场保留 |
| CI/CD 集成 | 必须支持批处理无头执行、标准退出码、报告与截图产物 |
| C# 扩展能力 | 必须支持自定义动作和 Page Object 封装 |

## 非目标

| 项目 | 结论 |
| --- | --- |
| uGUI 测试 | V1 不覆盖 |
| 移动真机测试 | V1 不覆盖 |
| 录制生成 YAML | V1 不覆盖 |
| 浏览器端远程执行 | V1 不覆盖 |

## 目标技术栈

| 组件 | 目标技术 | 说明 |
| --- | --- | --- |
| 语言 | C# | Unity 官方主语言 |
| 测试框架 | NUnit 3.x + Unity Test Framework | 统一测试入口与生命周期 |
| UI 测试底层 | `com.unity.ui.test-framework` | 提供窗口和交互模拟能力 |
| YAML 解析 | `YamlDotNet` 或 `VYaml` | 仅用于手写测试用例 |
| 截图 | `ScreenCapture.CaptureScreenshot` | 用于关键步骤和失败证据 |
| 报告 | Markdown + 文件 I/O | 生成轻量文本报告 |
| 可视化调试 | `EditorWindow` + IMGUI/Overlay | 构建 Headed 面板与高亮 |

## 总体分层架构

```text
User Layer
├─ YAML 用例
└─ C# 扩展动作 / Page Object

Parsing Layer
├─ YamlTestCaseParser
└─ ExecutionPlanBuilder

Engine Layer
├─ TestRunner
├─ StepExecutor
├─ ActionRegistry
└─ ElementFinder

Adapter Layer
├─ UnityUIFlowFixture<TWindow>
├─ EditorWindowUITestFixture<TWindow>
└─ PanelSimulator / UIToolkit Query

Presentation Layer
├─ HeadedTestWindow
└─ MarkdownReporter / ScreenshotManager
```

## 全局设计约束

1. 首版运行模式固定为 `EditMode`。
2. 所有公共 API 必须提供 XML 注释。
3. 不允许通过底层方法劫持破坏 Unity 编辑器稳定性。
4. 截图只允许在显式截图步骤和失败场景生成。
5. Headed 关闭时不得加载高亮绘制与调试事件订阅。
6. 无法从当前仓库验证的实现细节必须保留 `TODO(待确认)` 并登记到 `TODO.md`。
