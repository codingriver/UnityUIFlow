# UnityUIFlow 架构设计与技术约束

版本：1.1.0  
日期：2026-04-10  
状态：更新基线

## 项目背景

UnityUIFlow 的目标是为 `UIToolkit` 提供一套接近 Playwright headed 体验的 `Editor` 内自动化测试框架。

V1 固定范围为 `EditMode + EditorWindow + UIToolkit`。  
V1 不覆盖 `uGUI`、移动真机、录制回放和运行时玩家构建。

## 产品目标

| 目标 | 结论 |
| --- | --- |
| 声明式测试用例 | 必须支持 YAML 手写用例，并允许 QA 独立维护 |
| 真实交互模拟 | 必须接入 `com.unity.ui.test-framework`，作为点击、拖拽、悬停、滚动、窗口宿主能力的正式基线 |
| 高保真键盘与输入 | 必须接入基于 `com.unity.inputsystem` 的测试输入能力，用于 `press_key`、`type_text` 与快捷键回归验证 |
| Headed 调试体验 | 必须支持实时高亮、步进、失败暂停、现场保留 |
| CI/CD 集成 | 必须支持批处理无头执行、标准退出码、报告与截图产物 |
| C# 扩展能力 | 必须支持自定义动作和 Page Object 封装 |

## 非目标

| 项目 | 结论 |
| --- | --- |
| uGUI 测试 | V1 不覆盖 |
| 移动真机测试 | V1 不覆盖 |
| YAML 录制生成 | V1 不覆盖 |
| 浏览器端远程执行 | V1 不覆盖 |
| IME 组合输入完整仿真 | V1 不作为首版验收基线 |

## 目标技术栈

| 组件 | 目标技术 | 说明 |
| --- | --- | --- |
| 语言 | C# | Unity Editor 主语言 |
| 测试框架 | NUnit 3.x + Unity Test Framework | 统一测试入口与生命周期 |
| UI 交互仿真 | `com.unity.ui.test-framework` | 独立包，提供 `EditorWindowUITestFixture<T>` 与 `PanelSimulator`；`@6.3.0` 已添加到 `manifest.json` |
| 键盘/设备输入仿真 | `com.unity.inputsystem` 测试输入能力 | 提供 `InputTestFixture` 一类的高保真输入注入能力 |
| YAML 解析 | 当前实现为自研解析器 | 仓库当前未接入第三方 YAML 包 |
| 截图 | `ScreenshotManager` | 当前实现已优先输出真实窗口/屏幕截图，仅在全部链路失败时退回占位 PNG |
| 报告 | Markdown + JSON + 文件 I/O | 输出轻量文本报告与产物清单 |
| 可视化调试 | `EditorWindow` + UIToolkit Overlay | 构建 Headed 面板、高亮与批量运行器 |

## 当前仓库现状与目标差异

| 项目 | 当前仓库 | 目标基线 | 处理原则 |
| --- | --- | --- | --- |
| Unity 版本 | `6000.6.0a2` | 以当前仓库版本为实现基准，不回退 | 与官方测试包兼容性需要 PoC 验证 |
| `com.unity.test-framework` | `1.7.0`（NUnit 集成与测试运行器） | 已安装；负责 NUnit 测试生命周期与 TestRunner UI | 无 |
| `com.unity.ui.test-framework` | 已安装 `6.3.0` | 独立包，提供 `EditorWindowUITestFixture<T>` / `EditorWindowPanelSimulator` / `PanelSimulator` 官方交互仿真 | 已完成安装验证并接入主执行链 |
| 动作模拟 | `PanelSimulator` + `InputSystemTestFramework+UIToolkitBridge` + fallback 兼容链路 | `com.unity.ui.test-framework` / InputSystem 双栈驱动 | 当前实现为官方优先，fallback 仅用于非官方入口 |
| `type_text` 语义 | 默认官方 `PanelSimulator`；必要时回退到 InputSystem / 直接赋值 | 真实键盘输入链路 | 当前实现已具备官方默认路径，但 IME 等高阶输入仍不宣称完全覆盖 |
| Fixture 宿主 | `UnityUIFlowFixture<TWindow>` + `OfficialEditorWindowPanelSimulator` 桥接 | `EditorWindowUITestFixture<TWindow>` 直接继承属于可选增强路径 | 当前实现已完成官方宿主桥接 |
| 截图 | 真实窗口/屏幕截图优先 | 占位 PNG 仅作最终 fallback | 报告链路已保留，后续可继续补视觉 diff |

## 总体分层架构

```text
User Layer
|- YAML 用例
|- C# 自定义动作 / Page Object

Parsing Layer
|- YamlTestCaseParser
|- ExecutionPlanBuilder

Engine Layer
|- TestRunner
|- StepExecutor
|- ActionRegistry
|- ElementFinder

Adapter Layer
|- UnityUIFlowFixture<TWindow>
|- OfficialEditorWindowPanelSimulator   (当前官方宿主桥接，来自 com.unity.ui.test-framework@6.3.0)
|- EditorWindowUITestFixture<TWindow>   (可选增强路径，来自 com.unity.ui.test-framework@6.3.0)
|- Official UI Interaction Driver       (PanelSimulator，来自 com.unity.ui.test-framework@6.3.0)
|- InputSystem Keyboard Driver          (已接入，作为严格模式与补充链路)

Presentation Layer
|- TestRunnerWindow
|- HighlightOverlayRenderer
|- MarkdownReporter / ScreenshotManager
```

## 全局设计约束

1. 首版运行模式固定为 `EditMode + EditorWindow + UIToolkit`。
2. `com.unity.ui.test-framework@6.3.0`（独立包，提供 `EditorWindowUITestFixture<T>` 与 `PanelSimulator`）是 UnityUIFlow 宣称“真实 UI 交互仿真”能力的必需依赖；已添加到 `Packages/manifest.json`。
3. 基于 `com.unity.inputsystem` 的测试输入能力是 UnityUIFlow 宣称“高保真键盘输入”能力的必需依赖。
4. `type_text_fast` 继续保留为直接写值的快速路径，但不得作为真实用户输入能力的验收依据。
5. 在官方包接入完成前，当前 `SendEvent` / 直接赋值实现仅允许作为迁移期 fallback；文档必须显式标注“当前实现”与“目标基线”。
6. Headed、报告、Fixture、动作系统必须共享同一套宿主窗口与执行上下文，不允许为单个动作临时创建额外宿主窗口。
7. 无法从当前仓库验证的实现细节必须保留 `TODO(待确认)`，并登记到 `cocs/TODO.md`。

## 2026-04-11 架构现状修订

- YAML 解析现状已切换为 `YamlDotNet`，不再是“完全自研解析器”。
- 截图链路现状已升级为真实截图优先，不再是默认 2x2 占位 PNG。
- 自定义动作扫描现状已升级为“默认程序集白名单 + `.unityuiflow.json` 扩展”。
